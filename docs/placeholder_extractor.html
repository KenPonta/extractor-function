Image detection — the key trick

A shape counts as an image if it is a picture shape or its fill is a picture. Most extractors check only the first and report "no images" on decks whose charts are pasted as fills.

Case 1 · true picture shape

shape.shape_type == PICTURE → shape.image.blob

✓ Ordinary pasted images. The standard API works here.

Case 2 · picture fill on a shape

shape.fill.type == PICTURE → <a:blip r:embed> → related_part().blob

✗ standard API misses this — yet it is how this deck stores its charts. We follow the fill relationship to the embedded file.

Both read the original embedded file straight from the .pptx ZIP — no LibreOffice, no rendering. Only SVGs are rasterized (PyMuPDF), because the vision API needs raster.
